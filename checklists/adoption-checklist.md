# Adoption Checklist

## Minimum Setup

- [ ] `AGENTS.md` exists and tells the agent to use GitHub as the control plane.
- [ ] `github-harness-workflow` Skill is available to the agent.
- [ ] Demand Discussion template exists.
- [ ] Task issue template exists.
- [ ] Evidence comment template exists.
- [ ] The human knows the four review questions: direction, boundary, evidence, next step.

## First Loop

- [ ] First demand Discussion opened.
- [ ] Demand confirmation commit written.
- [ ] First task issue created.
- [ ] Agent restated goal, scope, acceptance, and evidence target.
- [ ] Evidence comment posted.
- [ ] Human review decision recorded.

## Scale Later

- [ ] PR template added.
- [ ] Board added for multiple issues.
- [ ] Labels added for status or priority.
- [ ] Automation added only after the manual loop is clear.
