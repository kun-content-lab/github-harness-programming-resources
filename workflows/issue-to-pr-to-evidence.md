# Workflow: Issue To PR To Evidence

Use this when the task requires file changes.

## Steps

1. Read the task issue.
2. Restate goal, scope, acceptance, and evidence target.
3. Make the smallest useful change.
4. Open a PR when reviewable files changed.
5. Fill the PR template.
6. Run verification that matches the artifact.
7. Post an evidence comment back to the issue.

## Output

- PR with change map and verification.
- Evidence comment linked to the issue.

## Stop Condition

Do not close the issue until the evidence comment is posted and reviewed.
