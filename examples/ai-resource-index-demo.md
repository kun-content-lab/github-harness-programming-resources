# 示例：AI 资料索引站

这个例子展示一个非代码项目如何使用最小 Harness Engine。

## 需求 Discussion

不要写：

> 我要做一个很厉害的 AI 资料网站。

应该写成：

> 这个站先服务想入门 AI 工作流的人；第一版只收 20 条高质量资料；只做分类、摘要、来源；不做登录，不做社区，不做推荐算法。

## 需求确认 commit 示例

```markdown
## 需求确认 commit

- 目标用户：想入门 AI 工作流、但不知道从哪些资料开始看的人。
- 第一版目标：做一个轻量资料索引站，帮助用户快速找到 20 条高质量资料。
- 本轮范围：资料分类、每条资料摘要、来源链接、适合人群、推荐阅读顺序。
- 明确不做：登录、社区、收藏、推荐算法、自动爬取、付费功能。
- 必要内容：资料标题、分类、摘要、来源、推荐理由、阅读难度。
- 验收标准：页面能展示 20 条资料；每条资料都有来源链接；用户能按分类浏览。
- 仍未确定：分类名称是否按“工具、工作流、案例、方法论”四类走。
- 建议拆出的 issue：整理资料清单、设计信息结构、生成首页原型、补证据截图。
```

## Task issue 示例

```markdown
## 任务目标

整理第一版 AI 资料索引站的 20 条资料清单。

## 来源

- 来自 Discussion：AI 资料索引站需求确认
- 对应需求确认 commit：第一版只收 20 条高质量资料；只做分类、摘要、来源

## 范围

### 本次要做

- 选出 20 条资料。
- 给每条资料补分类、摘要、来源链接、推荐理由、阅读难度。

### 本次不做

- 不做页面设计。
- 不做登录、社区、推荐算法。
- 不自动爬取全网资料。

## 验收标准

- [ ] 清单刚好 20 条。
- [ ] 每条都有来源链接。
- [ ] 每条都有 1-2 句摘要。
- [ ] 每条都有分类和阅读难度。
- [ ] 完成后提交 evidence comment。
```

## Evidence comment 示例

```markdown
## 状态

ready-for-review

## 这次完成了什么

- 整理了 20 条 AI 工作流入门资料。
- 每条资料补充了分类、摘要、来源、推荐理由和阅读难度。

## 证据在哪

| 证据 | 链接或位置 | 说明 |
|---|---|---|
| 资料清单 | docs/resources.md | 20 条资料完整列表 |
| 分类统计 | docs/resources.md#分类 | 四类资料数量 |

## 没做什么

- 没做网站页面。
- 没做自动推荐。
- 没做资料自动更新。

## 下一步

我的建议：可以 close 当前资料清单 issue，然后新开一个 issue 做首页信息结构。
```
