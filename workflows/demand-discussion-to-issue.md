# Workflow: Demand Discussion To Issue

Use this when the work starts as a loose idea.

## Steps

1. Open a demand Discussion.
2. Write the rough request.
3. Ask the agent to identify missing requirements.
4. Answer the missing questions.
5. Ask the agent to write a demand confirmation commit.
6. Review and edit the commit until it is accurate.
7. Split one narrow task issue from the commit.

## Output

- One demand confirmation commit.
- One or more task issues.

## Stop Condition

Do not implement until at least one task issue exists.
